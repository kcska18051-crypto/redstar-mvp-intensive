# Инструкция: Создание MVP

Ты создаёшь работающий MVP с живой базой данных. Это продукт, который можно показывать реальным пользователям.

## Входные данные

- PRD-файл — там описан продукт и его функции
- `1-2-MVP/prompts/landing-design.md` — инструкции по дизайну

## Главные принципы

### Что делаем

- Работающий продукт с живой базой данных
- CRUD-операции (создание, чтение, обновление, удаление)
- Красивый UI по инструкциям из `1-2-MVP/prompts/landing-design.md`
- Seed data — база сразу заполнена тестовыми данными

### Чего НЕ делаем

- Никакой авторизации
- Минимум зависимостей

## Технический стек

**Next.js (App Router)** + **Tailwind CSS** + **PostgreSQL** (на учебном VPS)

## Создание проекта

Проект создаётся в папке `1-2-MVP/product-mvp/` (то же имя, что в задании урока — без вложенных подпапок).

```bash
npx create-next-app@latest 1-2-MVP/product-mvp --typescript --tailwind --eslint --app --src-dir --no-import-alias --yes
cd 1-2-MVP/product-mvp
npm install pg @types/pg
```

## Где что работает (важно)

Приложение и база данных живут **на учебном VPS**. PostgreSQL слушает только локально на VPS (`localhost:5432`) и наружу не открыт — так безопаснее. Поэтому `DATABASE_URL` с `localhost` корректен: приложение обращается к БД с того же сервера.

Локальный `npm run dev` на твоём компьютере — только для быстрой проверки вёрстки; до БД на VPS он по `localhost` не достучится (и открывать БД наружу не нужно). Полноценно продукт с данными работает после деплоя на VPS.

## База данных (PostgreSQL)

Спроси у пользователя SSH-доступ к учебному VPS (IP-адрес, логин, и ключ или пароль — что выдал бот; лучше ключ). Затем сам:

1. Подключись к VPS по SSH
2. Установи PostgreSQL: `sudo apt update && sudo apt install -y postgresql`
3. Создай пользователя и базу данных
4. Создай `.env` файл с `DATABASE_URL` (например: `postgresql://user:password@localhost:5432/mydb`). Держи `.env` в `.gitignore` — секреты не должны попасть в git
5. Создай таблицы SQL-запросами на основе PRD
6. Заполни тестовыми данными (seed) — чтобы UI сразу показывал контент

## Дизайн

Следуй инструкциям из `1-2-MVP/prompts/landing-design.md`:
- Выбери смелое эстетическое направление
- Уникальная типографика (не Inter, не Arial)
- Анимации и микро-интеракции
- Никакого generic AI-дизайна

## Чек-лист готовности

Локально (быстрая проверка вёрстки):
- [ ] `npm run dev` запускается без ошибок
- [ ] UI выглядит уникально и красиво
- [ ] `npm run build` проходит успешно

На VPS (после деплоя — там работает продукт с данными):
- [ ] Данные сохраняются в PostgreSQL на VPS
- [ ] CRUD работает

## Деплой на VPS

Когда пользователь просит задеплоить — делай всё сам, не проси пользователя выполнять команды.

**Шаги:**

1. Собери проект локально: `npm run build`
2. Скопируй файлы на VPS:
   ```bash
   rsync -avz --exclude node_modules --exclude .git ./ user@VPS_IP:/home/user/product-mvp/
   ```
3. На VPS через SSH:
   ```bash
   cd /home/user/product-mvp && npm install --production
   npx pm2 stop product-mvp 2>/dev/null; npx pm2 start npm --name "product-mvp" -- start
   ```
4. Сообщи пользователю ссылку: `http://VPS_IP:3000`

> **При повторном деплое** — те же шаги. rsync обновит только изменённые файлы.

> Для учебного одноразового VPS мы намеренно упрощаем: голый HTTP, порт 3000, без отдельного пользователя/HTTPS/firewall. На боевом проекте так не делают — там HTTPS, отдельный непривилегированный пользователь, закрытые лишние порты и SSH по ключу (см. правила безопасности в воркшопе 5).

## Результат

Создай проект в папке `1-2-MVP/product-mvp/` с Next.js. После создания:

1. Запусти `cd 1-2-MVP/product-mvp && npm run dev`
2. Скажи: "Каркас MVP запущен локально на http://localhost:3000 (данные появятся на VPS после деплоя)"
